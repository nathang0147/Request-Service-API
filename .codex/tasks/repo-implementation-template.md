# Repo Implementation Task Template

Use this after the local parent agent has decided the task belongs in `Request-Service-API` and has narrowed the ownership set.

## Objective

- What specific change is being implemented?

## Owned Files

- Exact files this task may modify

## Assumptions

- What is assumed to already exist?
- What external contracts are being relied on?

## Non-Goals

- What should not be changed?

## Implementation Notes

- Target layer: `src/http/**`, `src/orchestration/**`, `src/integrations/**`, `src/domain/**`, `docs/**`, or bootstrap docs/config
- Keep the change small
- Avoid unrelated cleanup

## Validation

- Exact commands to run
- What each command proves

## Stop Conditions

- Missing repo scaffold
- Missing sibling-repo change
- Ambiguous ownership

## Handoff Output

- summary of the change
- files modified
- validation result
- unresolved risks
