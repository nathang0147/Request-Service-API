# Repo Entry Task Template

Use this when a controller above this repository routes a task into `Request-Service-API`.

## Objective

- What outcome is needed?

## Why This Repo Owns It

- Why does the work belong in `Request-Service-API` instead of a sibling repo?

## Parent Context

- What external context is the controller already providing?
- Which sibling repos or upstream systems matter?

## In Scope

- Exact files or folders expected to be touched

## Out of Scope

- Explicit non-goals
- Work that belongs in another repository

## Dependencies

- Required upstream inputs
- Schema or protocol references
- Blocking decisions outside this repo

## Success Criteria

- What must be true for the task to count as done?

## Validation Budget

- Structural checks
- Runtime checks if tooling exists

## Preferred Subagents

- `repo_mapper`
- `request_flow_specialist`
- `api_contract_guard`
- `cross_repo_dependency_scout`
- `verification_guard`

## Final Handoff Format

- changed files
- verification performed
- remaining dependencies or follow-ups
